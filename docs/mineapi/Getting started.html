Getting started

Hello! You are a developer? And you want use MineAPI?

It's simple!

First, we need Java 8 and Spigot API 1.9. Secondly, we have 2 solutions:

  • Use MineAPI without Gradle
  • Use MineAPI with Gradle

MineAPI without Gradle

You don't use Gradle or Maven? Ok and follow the instructions!
To use MineAPI, you need to download the latest development or release version and you have just to add the jar in the classpath.
Now we can develop with MineAPI!

MineAPI with Gradle

Yay, we want use Gradle! And Maven? Maven isn't compatible with the dependencies (Spigot server) because the POM file can't support this.
In short we need to use Gradle. Why Gradle, it's best than Maven because, it's newer, it can supports more instructions...
To develop, we need to open the Terminal or CMD. Whether you have Gradle in the Environment Variable we can use the command gradle.
After we have initialized the Gradle project, we modify the build.gradle and a some codes! 

apply plugin: 'java'
apply plugin: 'eclipse'
apply plugin: 'idea' // I recommend Intellij IDEA because it's have a good Gradle integration.


group = 'YourGroupHere' // Example for me: com.w67clement
version = '1.0-SNAPSHOT' // The version
archivesBaseName = 'PluginNameHere' // Name of you're plugin.


sourceCompatibility = 1.8 // We use Java 8


repositories {
    maven {
        url "https://dl.bintray.com/w67clement/maven"
    }
    // It's for NMS
    ivy {
        url "http://www.tcpr.ca/files/"
        layout 'pattern', {
            artifact '[module]/[module]-[revision].[ext]'
        }
    }
    mavenCentral();
}


dependencies {
    compile 'com.w67clement.mineapi:mineapi:LatestVersionHere' // MineAPI add Spigot automatically
}

For the LatestVersionHere, I have one example: 2.2.1 or 2.3.0, for the versions available, look at: https://bintray.com/w67clement/maven/MineAPI/view
Wow it's simple! Now you can use MineAPI in your code!
